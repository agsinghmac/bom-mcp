# Data Model: ESP Selection and BOM Readiness Skill Resource

## Entity: SkillResource

- Description: MCP-exposed instruction resource discoverable via `skill://` URI.
- Fields:
  - `uri` (string, required): Canonical skill URI, `skill://esp-selection-bom-readiness`
  - `name` (string, required): Human-readable title
  - `version` (string, required): Skill instruction version
  - `description` (string, required): Short purpose statement
  - `instruction_sections` (array<InstructionSection>, required): Ordered execution guidance
  - `required_tools` (array<string>, required): Existing MCP tool names agent should use
  - `output_template` (OutputTemplate, required): Expected final report structure
- Validation:
  - URI must start with `skill://`
  - At least one instruction section must exist
  - Required tools must reference existing server tools

## Entity: OperationalRequirement

- Description: Input constraints for recommendation workflow.
- Fields:
  - `required_flow_m3d` (number, required, > 0)
  - `max_power_kw` (number, required, > 0)
  - `frequency_hz` (number, required, allowed set from deployment context)
- Validation:
  - All fields required
  - Values must be positive and within configured engineering bounds

## Entity: EspCandidate

- Description: Candidate ESP model evaluated against constraints.
- Fields:
  - `esp_id` (string, required)
  - `series` (string, optional)
  - `fit_rank` (integer, required)
  - `fit_summary` (string, required)
  - `flow_delta` (number, required)
  - `power_headroom_kw` (number, required)
  - `frequency_match` (boolean, required)
- Validation:
  - Rank must be unique per assessment
  - Fit summary must include primary/binding constraint statement

## Entity: BomReadinessReport

- Description: Readiness assessment for one recommended ESP.
- Fields:
  - `esp_id` (string, required)
  - `critical_parts_total` (integer, required)
  - `critical_parts_high_risk` (integer, required)
  - `critical_availability_percent` (number, required)
  - `field_ready` (boolean, required)
  - `availability_breakdown` (object, required): counts by GREEN/YELLOW/RED/BLOCKED
  - `procurement_alerts` (array<PartReadinessAlert>, required)
- Validation:
  - `critical_availability_percent = ((total - high_risk)/total) * 100` when total > 0
  - `field_ready = (critical_availability_percent >= 80)`

## Entity: PartReadinessAlert

- Description: Part-level risk item for procurement review.
- Fields:
  - `part_number` (string, required)
  - `part_name` (string, required)
  - `is_critical` (boolean, required)
  - `availability_flag` (enum, required): GREEN | YELLOW | RED | BLOCKED
  - `risk_reason` (string, required)
  - `urgency` (enum, required): LOW | MEDIUM | HIGH | BLOCKING
- Validation:
  - RED/BLOCKED + critical part must map to HIGH or BLOCKING urgency

## Entity: AssessmentOutput

- Description: Final combined result generated by agent following skill instructions.
- Fields:
  - `recommendations` (array<EspCandidate>, required)
  - `bom_readiness_reports` (array<BomReadinessReport>, required)
  - `summary` (string, required)
- Validation:
  - At least one recommendation when matching candidates exist
  - One readiness report per recommendation
